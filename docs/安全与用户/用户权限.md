# 用户与权限

本文档归档 **j2agent** 的用户数据模型、登录会话、角色鉴权与用户管理 REST API。邮箱自助注册见 [邮箱注册机制.md](./邮箱注册机制.md)。

## 用户模型

### 数据表

库表 `j2agent.user`（[`V0_0__schemas.sql`](../../j2agent-server/src/main/resources/sql/migration/mysql/zh_CN/V0_0__schemas.sql)）：

| 列 | 类型 | 说明 |
|----|------|------|
| `id` | `char(32)` PK | UUIDv7（[`UUIDv7Utils`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/utils/UUIDv7Utils.java)） |
| `username` | `varchar(64)` NOT NULL | 登录名 |
| `password_hash` | `char(64)` NOT NULL | `SHA256(userId + password)` |
| `create_time` | `bigint` | 创建时间戳 |
| `role` | `int` | `1` 管理员，`2` 普通用户 |
| `email` | `varchar(128)` NULL | 邮箱；唯一索引 `uq_user_email` |

密码算法见 [`UserUtil`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/utils/UserUtil.java)：`getPasswordHash(userId, password)` / `verifyPassword`。

### 内置管理员

Flyway [`V0_1__init_data.sql`](../../j2agent-server/src/main/resources/sql/migration/mysql/zh_CN/V0_1__init_data.sql) 插入：

- `username` = **`aiadmin`**
- `role` = `1`（管理员）
- 固定 `id`：`AZ5EsB1JcMqXv6z1FcfEUw`

[`UserService`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/service/security/UserService.java) 中常量 `BUILTIN_ADMIN_USERNAME = "aiadmin"`，对该用户：

- 不可删除
- 不可修改角色
- 其他管理员不可为其重置密码（`updateUserPassword` 中 `ensureMutableUser`）

## 角色与鉴权

### 角色枚举

[`SessionBo.RoleEnum`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/model/security/SessionBo.java)：

| 枚举 | 值 | 说明 |
|------|-----|------|
| `ADMIN` | 1 | 管理员 |
| `USER` | 2 | 普通用户 |

**数值越小权限越高。** 判断能否访问某接口要求：

```text
session.role.value <= requiredRole
```

例如：要求 `ADMIN(1)` 时，仅 `role=1` 通过；要求 `USER(2)` 时，管理员与普通用户均可通过。

`session.isAdmin()` 等价于 `hasAccess(ADMIN)`。

### @RequiredRole 注解

[`RequiredRole`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/config/annotation/RequiredRole.java) 标注在 Controller 类或方法上：

| 常量 | 值 | 含义 |
|------|-----|------|
| `RequiredRole.ADMIN` | 1 | 仅管理员 |
| `RequiredRole.USER` | 2 | 已登录即可（含管理员） |

未标注时，[`LoginInterceptor`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/interceptor/LoginInterceptor.java) 默认按 `USER(2)` 校验。

### LoginInterceptor

配置于 [`InterceptorConfig`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/config/InterceptorConfig.java)：

- 拦截：`/v*/**`
- 排除：`/v*/auth/**`（登录、验证码、邮箱注册等公开接口）

流程：

1. [`ApiLoginChecker`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/interceptor/ApiLoginChecker.java) 从 Cookie `SESSION-{serverPort}` 读取 sessionId，调用 `LoginService.validateSession` 并 `bindSession`。
2. 未登录：**401**，并 `clearSession`。
3. 已登录但角色不足：**403**，并 `clearSession`。
4. 请求结束：`afterCompletion` 中 `clearSession`，避免线程池复用导致会话串号。

### 路径与权限矩阵

| 区域 | 路径前缀 | 需要登录 | 典型角色要求 |
|------|----------|----------|----------------|
| 认证公开 | `/v1/auth/**` | 否 | — |
| 业务 REST | `/v1/rest/**` | 是 | 见各接口 `@RequiredRole` |
| 用户管理 | `/v1/rest/j2agent/users` 等 | 是 | 见下节 |

**类级别 `@RequiredRole(ADMIN)` 的控制器**（整类接口仅管理员）：

| 控制器 | 说明 |
|--------|------|
| `KnowledgeController` | 知识库 |
| `PropertyController` | 系统属性（含邮箱注册开关等） |
| `McpController` | MCP |
| `EmbeddingsController` | 向量 |
| `AgentPluginController` | Agent 插件 |
| `HealthCheckController` | 健康检查 |

**未标注或方法级标注**：如 `ChatController` 等，默认 **已登录普通用户** 即可；`UserController` 在方法上区分 ADMIN / USER。

服务层 [`UserService`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/service/security/UserService.java) 对管理操作另有 `requireAdmin()`，与拦截器形成 **双重校验**（拦截器为主，Service 为防御性检查）。

## 登录与会话

### 登录

- **接口**：`POST /v1/auth/j2agent/login`（[`LoginController`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/controller/LoginController.java)）
- **请求体**：`username`、`password`、`validateCode`、`hash`（滑动验证码，与注册发码相同机制）
- **成功**：设置 Cookie `SESSION-{serverPort}`，`httpOnly`，`path=/`，值为 `sessionId`
- **失败**：401

### 会话存储

[`LoginService`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/service/security/LoginService.java) 使用进程内 `ConcurrentHashMap<String, SessionBo>`：

| 行为 | 说明 |
|------|------|
| 创建会话 | 登录成功时 `expireTime = now + 24h` |
| 活跃续期 | 每次 `validateSession` 成功将 `expireTime` 设为 `now + 30min` |
| 定时清理 | 每 5 分钟移除已过期 session |
| 登出 | `logout(sessionId)` 从 Map 移除 |
| 强制失效 | 删除用户、改角色、改密码后 `invalidateUserSessions(userId)` |

`SessionBo` 字段：`sessionId`、`userId`、`username`、`role`、`expireTime`。

### 当前会话查询

- **接口**：`GET /v1/rest/j2agent/session`（[`SessionController`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/controller/SessionController.java)）
- **响应**：`SessionInfoDto`（`userId`、`username`、`role`）
- **未登录**：401

### 登出

- **接口**：`GET /v1/auth/j2agent/logout`

## 用户管理 API

实现：[`UserController`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/controller/UserController.java) + [`UserService`](../../j2agent-server/src/main/java/com/nms/platsvc/ai/center/service/security/UserService.java)。

OpenAPI：[`openapi-interface.yaml`](../../j2agent-sdk/j2agent-model/src/main/resources/openapi-interface.yaml)（`User` tag）。

| 方法 | 路径 | @RequiredRole | 说明 |
|------|------|---------------|------|
| GET | `/v1/rest/j2agent/users` | ADMIN | 用户列表，`create_time DESC, username ASC` |
| POST | `/v1/rest/j2agent/users` | ADMIN | 创建用户；body：`username`、`password`、可选 `role`（默认 USER） |
| DELETE | `/v1/rest/j2agent/users?user-id=` | ADMIN | 删除用户；内置 `aiadmin` 不可删 |
| PUT | `/v1/rest/j2agent/users/role` | ADMIN | 修改角色；`userId` + `role`（仅 1 或 2） |
| PUT | `/v1/rest/j2agent/users/password` | USER | 修改密码；见下表 |

请求体 `UserPasswordUpdateRequestDto`：`userId`（可选）、`newPassword`。

| 操作者 | 目标 | 规则 |
|--------|------|------|
| 普通用户 | 自己（`userId` 省略或等于当前用户） | 允许修改 |
| 普通用户 | 他人 | **403** |
| 管理员 | 自己 | 允许修改 |
| 管理员 | 他人 | 允许重置；**不可**修改 `aiadmin` |

改密成功后对该用户执行 `invalidateUserSessions`。

删除用户、修改角色后同样会使该用户所有会话失效。

## 用户创建途径对比

| 途径 | 接口 / 入口 | 角色 | 说明 |
|------|-------------|------|------|
| 迁移脚本 | SQL | ADMIN | 仅 `aiadmin` |
| 管理员创建 | `POST .../users` | 可指定 1 或 2 | 需管理员登录 |
| 邮箱自助注册 | `POST .../register` | 固定 USER(2) | 见 [邮箱注册机制.md](./邮箱注册机制.md) |

自助注册用户可用邮箱（或注册时指定的 `username`）登录；`email` 字段写入 `user.email`。
