# 离线镜像打包

脚本位置：[`docker/package_offline.sh`](../../docker/package_offline.sh)

用于在无外网的目标机上部署：在**有网络的机器**上拉取/构建 Compose 所需镜像并导出为 `.tar`，再到目标机 `docker load` 后 `docker compose up`。

## 使用步骤

1. 进入 `docker` 目录，确保已存在 `docker-compose.yml` 及正确的 `.env`（或先 `cp .env.example .env`）。

2. 赋予执行权限并运行：

   ```bash
   cd docker
   chmod +x package_offline.sh
   ./package_offline.sh
   # 或指定 compose 文件：./package_offline.sh docker-compose.yml
   ```

3. 按提示选择：

   - 目标架构：`linux/amd64` 或 `linux/arm64`
   - 要打包的镜像序号（回车表示全部）
   - 是否跳过全局拉取（默认优先使用本地已有镜像）
   - 导出方式：单个合并 `images_offline_<arch>.tar`，或分文件目录 `images_offline_<arch>_dir/`

4. 将生成的 tar 或目录拷贝到离线服务器，按脚本结束时的提示导入，例如：

   ```bash
   docker load -i images_offline_amd64.tar
   cd docker
   docker compose up -d
   ```

## 注意事项

- **j2agent** 若在本地 `docker compose build` 过，会包含在镜像列表中；离线机需已有同名 tag 或通过 save 一并导出。
- 离线环境仍需准备 **宿主机数据目录**（`dist`、`knowledge-repo` 等），镜像不包含前端与知识库内容。
- MySQL 首次启动会按 `.env` 初始化库名与 root 密码；数据目录若已存在则不会重复初始化。
