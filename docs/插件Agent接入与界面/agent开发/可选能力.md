# 可选能力

本文简要说明 Agent 除 Tool / Skill / MCP 外的常见扩展：**RAG**、**热门问题**、**深度思考**、**自定义拦截器**。

## 1. RAG（检索增强生成）

当 Agent 需要基于 **Milvus 知识库** 回答问题时，override **`buildDocumentRetriever()`** 返回 Spring AI `DocumentRetriever`。

基类 `buildAgent()` 会在 retriever 非空时自动挂载 `RetrievalAugmentationAdvisor`（混合检索 + 上下文注入）。

### 1.1 知识库准备

知识入库、collection 命名、分片规则见 [RAG 机制](../../RAG机制/README.md)。对话侧使用的 collection 名称须与知识库同步产物一致。

### 1.2 Retriever 实现

推荐继承平台 **`AbstractCollectionKbRetriever`**，绑定 collection 名称：

```java
package com.nms.prodplugin.ai.center.demo;

import com.nms.platsvc.ai.center.rag.AbstractCollectionKbRetriever;
import com.nms.platsvc.ai.center.service.rag.retrieval.Retriever;
import org.springframework.stereotype.Component;

@Component
public class DemoKbRetriever extends AbstractCollectionKbRetriever {

    public DemoKbRetriever(Retriever retriever) {
        super(retriever);
    }

    @Override
    protected String boundCollection() {
        // 与 knowledge-repo 入库后的 Milvus collection 名一致
        return "demo_knowledge";
    }
}
```

### 1.3 Agent 挂载

```java
@Component
@RequiredArgsConstructor
public class DemoAgent extends AiAgent {

    private final DemoKbRetriever demoKbRetriever;

    @Override
    protected DocumentRetriever buildDocumentRetriever() {
        return demoKbRetriever;
    }
}
```

需 import `org.springframework.ai.rag.retrieval.search.DocumentRetriever`。

### 1.4 行为说明

| 项 | 说明 |
|----|------|
| 检索时机 | 每轮用户消息经 RAG Advisor 触发（React 工具循环中会自动跳过重复检索） |
| 失败降级 | Milvus 检索失败时注入 fallback 文档，**不中断对话** |
| 空结果 | 无命中分片时返回空列表，模型基于对话上下文回答 |
| 与 LLM 配置关系 | RAG 不挂在 LLM 提供商条目上，完全由 Agent 代码决定 |

## 2. 热门问题（qa-template）

空会话时在输入框上方展示快捷问题，需同时满足：

1. 插件 JAR 内放置 **`qa-template.json`**
2. override **`isQaTemplateEnabled()`** 返回 `true`

### 2.1 资源格式

`src/main/resources/qa-template.json`：

```json
{
  "zh_CN": [
    "演示 Agent 能做什么？",
    "如何使用 echo 工具？",
    "如何查看当前时间？"
  ],
  "en_US": [
    "What can the demo agent do?",
    "How do I use the echo tool?",
    "How do I get the current time?"
  ]
}
```

### 2.2 Agent 代码

```java
@Override
public boolean isQaTemplateEnabled() {
    return true;
}
```

### 2.3 API 与前端

| 接口 | 说明 |
|------|------|
| `GET /v1/rest/j2agent/agents` | `AgentInfoDto.showHotQuestions=true` |
| `GET /v1/rest/j2agent/qa-template?agent-id=demo_agent&limit=5` | 从 Agent ClassLoader 随机抽取问题 |

UI 约定见 [Agent-UI 交互机制 3.6](../../agent-ui交互机制/README.md)。

## 3. 深度思考（Thinking）

深度思考控制模型是否启用 extended thinking（Anthropic / Ollama 等）。

### 3.1 优先级（高 → 低）

1. 单轮 WebSocket 请求 **`ChatRequestDto.thinkingMode`**（`provider_default` / `on` / `off`）
2. Agent 默认 **`getThinkingOverride()`**
3. 管理端 LLM 配置 **`thinkingMode`**

### 3.2 Agent 默认策略

```java
import com.nms.platsvc.ai.center.service.llm.agent.AgentThinkingOverride;

@Override
public AgentThinkingOverride getThinkingOverride() {
    // 示例：本 Agent 默认关闭深度思考
    return AgentThinkingOverride.OFF;

    // 其它可选值：
    // USE_PROVIDER_DEFAULT — 不覆盖，沿用 LLM 配置
    // PROVIDER_DEFAULT — 强制不传 thinking 参数
    // ON — 强制开启
}
```

详见 [LLM 提供商配置 1.2](../../LLM提供商配置/README.md)。

## 4. 自定义拦截器

`AiAgent` 默认挂载三条拦截器（工具错误兜底、工具 UI、Skill UI）。子类可 override **`buildInterceptors()`** 追加自定义 `Interceptor`：

```java
import com.alibaba.cloud.ai.graph.agent.interceptor.Interceptor;

@Override
protected Interceptor[] buildInterceptors() {
    return new Interceptor[] {
            agentUiToolEventInterceptor,
            agentUiSkillLoadToolInterceptor,
            myCustomInterceptor   // 追加
    };
}
```

**注意**：

- `AgentToolErrorReturnInterceptor` 由 `buildEffectiveInterceptors()` **强制保留**，无需（也不应试图移除）手动加入。
- 自定义拦截器应谨慎修改工具调用链路，避免与 UI 事件拦截器冲突。
