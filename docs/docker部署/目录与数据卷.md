# 目录与数据卷

Compose 通过环境变量 `J2AGENT_VOLUMES_PATH`（默认 `/opt/j2agent`）将宿主机目录挂载进容器。路径分隔符由 `DIR_PATH_SEPARATOR` 控制（Linux 为 `/`）。

## j2agent 服务挂载

| 宿主机路径 | 容器路径 | 用途 |
|------------|----------|------|
| `${J2AGENT_VOLUMES_PATH}/ui` | `/opt/j2agent/volume/ui` | 前端 SPA 构建产物（index.html、assets/） |
| `${J2AGENT_VOLUMES_PATH}/logs` | `/opt/j2agent/volume/logs` | 应用日志、OOM heap dump |
| `${J2AGENT_VOLUMES_PATH}/knowledge-repo` | `/opt/j2agent/volume/knowledge-repo` | Markdown/AsciiDoc 知识库仓库 |
| `${J2AGENT_VOLUMES_PATH}/plugins/agents` | `/opt/j2agent/volume/plugins/agents` | 智能体插件 JAR |

配置项对应关系（`application-product.yaml`）：

- `com.nms.ai.knowledge.repo.root-path` → `knowledge-repo`
- `com.nms.ai.plugin.path` → `plugins/agents`
- `spring.web.resources.static-locations` → `file:/opt/j2agent/volume/ui/`

## 依赖服务数据目录

| 服务 | 宿主机路径（示例） | 说明 |
|------|-------------------|------|
| MySQL | `${J2AGENT_VOLUMES_PATH}/mysql/data` | 数据文件 |
| MySQL | `.../mysql/conf/my.cnf` | 自定义配置（需事先准备） |
| MySQL | `.../mysql/run` | socket 目录 |
| Redis | `${J2AGENT_VOLUMES_PATH}/redis/data` | AOF 持久化 |
| etcd | `${J2AGENT_VOLUMES_PATH}/milvus/volumes/etcd` | Milvus 元数据（外置 etcd） |
| Milvus | `${J2AGENT_VOLUMES_PATH}/milvus/volumes/milvus` | 向量与本地存储数据 |
| Milvus | `${J2AGENT_VOLUMES_PATH}/milvus` | 可选覆盖配置 `user.yaml` |

## 推荐初始化目录

```bash
BASE="${J2AGENT_VOLUMES_PATH:-/opt/j2agent}"
mkdir -p "$BASE"/{dist,logs,knowledge-repo,plugins/agents}
mkdir -p "$BASE"/mysql/{data,conf,run}
mkdir -p "$BASE"/redis/data
mkdir -p "$BASE"/milvus/volumes/{etcd,milvus}
```

## 注意事项

1. **不要删除并重建已挂载的顶层目录**（如 `rm -rf dist && mkdir dist`），否则 bind mount 可能仍指向旧 inode，容器内看不到新文件。应在**已有目录内**同步文件，见 [前端静态资源更新.md](./前端静态资源更新.md)。
2. 后端 JAR/镜像**不包含** `dist`（assembly 已排除），首次部署必须自行放入前端产物。
3. 修改 `.env` 中的 `J2AGENT_VOLUMES_PATH` 后，需将数据迁移到新路径或保持与原挂载一致。
