# 静态文件展示机制

本文档固化 Markdown / AsciiDoc 知识库中图片等静态文件的展示机制。知识库目录与同步见 [知识库同步](知识库同步.md)。

## 1. 基本原则

- 图片等静态文件不参与文档分片。
- 静态文件不会写入 Milvus。
- 静态文件通过文件读取接口在问答结果中展示。
- 文档分片入库前，会将 `answer` 中的图片地址改写为可直连访问的 HTTP URL。

## 2. 文档写法

推荐在 Markdown 中使用相对路径引用图片：

```markdown
### 登录页入口
登录页如下：

![登录页](./images/login.png)
```

AsciiDoc 中也可以使用块图片语法，入库时会转换为 Markdown 图片语法供前端渲染：

```asciidoc
=== 登录页入口
登录页如下：

image::./images/login.png[登录页]
```

也支持：

```markdown
![流程图](images/login.png)
![上级目录图片](../shared/login.png)
![外链图片](https://example.com/login.png)
```

推荐目录结构：

```text
knowledge-repo/
  product/
    info.json
    faq.md
    manual.adoc
    images/
      login.png
```

## 3. 同步时的 URL 改写

文档分片入库前会改写 `answer` 中的图片地址：

- `http://`、`https://` 外链保持不变。
- 已经是 `/v1/rest/j2agent/file/repo/**` 的地址保持不变。
- 历史 `/v1/rest/j2agent/file/static/**` 地址会转为 repo 直链。
- 其他相对路径会以当前文档文件所在目录为基准解析，并转为 `/v1/rest/j2agent/file/repo/**`。
- 路径会逐段 URL 编码，支持中文文件名和空格。

例如文件 `product/faq.md` 或 `product/manual.adoc` 中写：

```markdown
![登录页](./images/登录 页.png)
```

入库后的答案会变成：

```markdown
![登录页](/v1/rest/j2agent/file/repo/product/images/%E7%99%BB%E5%BD%95%20%E9%A1%B5.png)
```

前端按 Markdown 渲染答案时，浏览器会直接请求该 URL 展示图片。

## 4. 文件读取接口

静态资源读取由 `FileController` 提供：

- `GET /v1/rest/j2agent/file/repo/**`：按知识库根目录相对路径读取文件。
- `GET /v1/rest/j2agent/file/static/**`：兼容历史静态路径；当前也只从知识库仓库查找。

读取安全规则：

- repo 直链会将请求路径解析到 `root-path` 下，越界路径会被拒绝。
- 若精确相对路径不存在，服务会在知识库内按相对路径后缀或文件名兜底搜索。
- 资源不存在返回 `404`。

Content-Type 规则：

- `.jpg` / `.jpeg`：`image/jpeg`
- `.png`：`image/png`
- `.gif`：`image/gif`
- `.pdf`：`application/pdf`
- `.txt`：`text/plain`
- 其他后缀：`application/octet-stream`

## 5. 资源组织建议

- 图片与知识库文档放在同一知识子树内，优先使用相对路径。
- 避免不同目录下使用完全相同的图片文件名；虽然服务有文件名兜底搜索，但重名时命中结果不保证符合预期。
- 需要迁移目录时，同步调整文档中的相对路径。
- 不建议在文档中直接写历史 `/file/static/**` 地址；新内容统一使用相对路径或 `/file/repo/**`。
