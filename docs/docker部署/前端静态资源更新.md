# 前端静态资源更新

生产环境「热更新」指：**替换宿主机 `dist` 后，无需重启 `j2agent` 容器**，浏览器即可加载新版本（非 Vite 开发服务器 HMR）。

## 架构简述

```
宿主机 ${J2AGENT_VOLUMES_PATH}/ui
    → bind mount → 容器 /opt/j2agent/volume/ui
    → Spring Boot (profile=product) file:/opt/j2agent/volume/ui/
    → 浏览器请求 /、/index.html、/assets/*
```

后端打包**故意排除** `dist/**`（见 `assembly-internal.xml`），前端由运维单独发布到挂载目录。

## 正确更新步骤（SOP）

1. 确认 `.env` 中 `J2AGENT_VOLUMES_PATH` 与实际操作目录一致。

2. 确认容器使用 `product` profile：

   ```bash
   docker compose exec j2agent sh -c 'echo $SPRING_OPTIONS'
   # 应包含 --spring.profiles.active=product
   ```

3. **在已有 `dist` 目录内同步**，不要删除挂载点目录本身：

   ```bash
   BASE="${J2AGENT_VOLUMES_PATH:-/opt/j2agent}"
   rsync -av --delete /path/to/new-frontend-dist/ "${BASE}/ui/"
   ```

   或使用 tar 解压到**已有**目录：

   ```bash
   tar -xf frontend-dist.tar.gz -C "${BASE}/ui"
   ```

4. **禁止**以下做法（会导致必须重启容器才生效）：

   ```bash
   # 错误示例：删除整个 dist 后重建
   rm -rf "${BASE}/ui"
   mkdir "${BASE}/ui"
   cp -r new-dist/* "${BASE}/ui/"
   ```

   原因：bind mount 绑定的是目录 inode；宿主机 `rm -rf dist` 后，容器内仍可能挂到已删除的旧目录。

5. 验证（**无需** `docker compose restart`）：

   ```bash
   docker compose exec j2agent stat /opt/j2agent/volume/ui/index.html
   curl -I "http://127.0.0.1:${J2AGENT_PORT}/index.html"
   ```

   宿主机与容器内文件的 mtime/size 应一致；`curl` 可见最新内容。

6. 浏览器侧：若仍像旧版，使用无痕窗口或强制刷新；`index.html` 已配置 `Cache-Control: no-cache`（product 环境），带 hash 的 `assets/*` 可长期缓存。

## 常见原因排查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 重启容器后才生效 | 删除并重建了 `dist` 挂载点目录 | 改用 `rsync` 进已有目录；必要时 `docker compose restart j2agent` 一次后按 SOP 维护 |
| 替换 dist 完全无效 | 未启用 `product` profile | 检查 `SPRING_OPTIONS` |
| 替换 dist 完全无效 | 改错路径（未改 `J2AGENT_VOLUMES_PATH/ui`） | 对照 compose 卷挂载 |
| 仅页面逻辑旧、资源 URL 新 | 浏览器缓存 `index.html` | 强刷 / 无痕；确认响应头 no-cache |
| 宿主机已更新、容器内 stat 不变 | Docker Desktop 卷延迟或挂载点问题 | `docker compose restart j2agent`；Linux 上检查是否误删挂载目录 |
| 与知识库图片混淆 | 更新的是 `knowledge-repo` 而非 `dist` | 知识库走 `/v1/rest/j2agent/file/repo/**`，见 [静态文件展示机制](../RAG机制/静态文件展示机制.md) |

## 应用侧配置（product）

`application-product.yaml` 已设置：

- `spring.web.resources.static-locations: file:/opt/j2agent/volume/ui/`
- `spring.web.resources.cache.period: 0`
- `spring.web.resources.chain.cache: false`

并对 `/`、`/index.html` 增加 `Cache-Control: no-cache`（`ProductIndexCacheControlFilter`）。

修改上述 YAML 或过滤器代码后，需**重建并重启** `j2agent` 容器一次；日常仅换前端文件不需要重启。

## 本地开发对比

| 环境 | 静态资源位置 |
|------|----------------|
| 本地默认 `application.yaml` | `classpath:/ui/` |
| Docker `product` | `file:/opt/j2agent/volume/ui/` |

本地将前端 build 到 `j2agent-starter/src/main/resources/ui/` 可随 classpath 加载；Docker 生产务必使用卷挂载目录。
